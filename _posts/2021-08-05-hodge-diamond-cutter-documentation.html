It was a tedious task, but the Hodge diamond cutter now has

  • online documentation (check it out!)
  • documentation as a pdf
  • unit tests

Using the library

It's enough to have the file diamond.py (pay attention to the file extension) somewhere where Sage can find it via load("diamond.py") and then you are good to go.

Unit tests

Having unit tests is very convenient to test regressions, and to showcase the functionality. Almost every method and construction now has (multiple) examples showcasing how to use it, and the library in general.

By typing sage -t diamond.py Sage automatically executes all the code in the examples section in the documentation, and reports (as of today) that 

Doctesting 1 file.
